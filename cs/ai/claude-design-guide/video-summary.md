## 한 줄 요약

Anthropic Claude Design 디렉터 Jenny Wen이 "기존 디자인 프로세스는 죽었다"고 선언한 지 한 달 만에 출시된 Claude Design은, **디자인 시스템 자동 생성 + Figma 임포트 + 4가지 수정 수단**으로 디자인 SW 시장을 흔들고 있다.

## 챕터별 요약

### 00:00 ~ 00:18 — Hook: Jenny Wen 영어 인용

> "Hey, you know this design process that designers have been taught — where you go and do a bunch of research and discovery, and then you diverge, you converge, diverge, converge — it's like this process that we sort of treated as gospel and tried so hard to preserve. We trust the process. **Dead.**"

영상은 이 인용으로 시작한다. 디자이너들이 복음처럼 신봉해 온 발산·수렴 사이클이 끝났다는 메시지다.

### 00:19 ~ 01:33 — 인트로

- 인용한 사람은 **Jenny Wen**: Anthropic의 Claude Design 총괄. 직전에는 **Figma에서 FigJam을 성공시킨 디자인 디렉터**였다.
- 즉, **Figma의 전성기를 이끌었던 에이스가 직접** "기존 디자인 프로세스는 끝났다"고 선언한 것.
- 그녀가 그렇게 말한 한 달 뒤, Anthropic은 **Claude Design을 출시**했다.
- **시장 충격**:
  - Figma 주가 **−7%**
  - Adobe 동반 하락
  - Wix 동반 하락
  - 월가가 디자인 SW 업계 가치를 그날 하루 만에 다시 평가했다는 표현
- 발표자의 평가: **"누구나 꽤 좋은 품질의 디자인 시스템을 너무 쉽게 만들 수 있게 된 점"**이 진정한 핵심이라고 본다.

### 01:33 ~ 04:18 — 개요 및 예시 갤러리

- 클로드 디자인은 **데스크탑 앱 미지원**, **웹 브라우저 전용**.
- 좌측 사이드바의 **Design** 메뉴로 진입.
- 4개 탭: **Prototype / Slidedeck / Prompt Templates / 새 유형**.
- 화면 상단의 **'플(Examples)'** 항목에서 결과물 예시 갤러리 확인 가능:

| 예시 | 특징 |
|------|------|
| 우주의 규모/천체 거리 | 스프라이트 기반 애니메이션 + 텍스트 결합 |
| 인터랙티브 그래픽 | 올가닉/추상적인 움직임 |
| 5종 쉐이더 | 마우스 호버에 실시간 반응 |
| 3D 트럼프 카드 | 호버 시 틸팅 + 무지갯빛 효과, 랭크/문양 변경 가능 |
| 계산기 / 자전거 회원가입 / 텍스트 스트리밍 | 일반 UI 패턴 |
| 3D 회전 지구본 | WebGL 기반 |
| 텍스트 파티클 효과 | 단어별 적절한 파티클이 적용된 타이포그래피 아트 |

> 발표자 메모: 효과 이름을 기억해 뒀다가 나중에 프롬프트로 입력하면 그 효과를 자기 디자인에 적용할 수 있다.

### 04:19 ~ 08:38 — 디자인 시스템 만들기 (Spotify 사례)

- 비교: 구글의 **Stitch**가 design.md 파일로 디자인 시스템을 정의하는 방식과 유사.
- 사용 자료: GitHub의 **awesome-design-md** 레포 — Apple, BMW, IBM, Coinbase, **Spotify** 등 글로벌 브랜드의 design.md 모음.
- Spotify design.md를 가져와 적용:
  - Company Name: "Spotify Design System"
  - GitHub URL 입력 시 외부 레포 접근 실패 케이스 발생 → **노트 필드에 design.md 내용 직접 붙여넣기**로 우회
  - 에셋: 로고 파일 + Spotify 앱 화면 스크린샷
- **Continue → Generate** → 약 5분 소요
- **Need Review 18개 항목** 표시:
  - 브랜드 컬러 (Spotify Green, Near Black, Dark Surface, Mid Dark)
  - Surface 컬러 / 텍스트 / 보더 / 타입 스펙 / 버튼 / Radius / Spacing / Elevation / Controls / Input/Filter / Playlist Card / Badge & Tag / Playing Bar / Logo / Icon
  - 각 항목별 **Looks Good** / **Needs Work** 선택
  - Needs Work 시 수정 방향을 프롬프트로 전송
- **Publish 토글**로 디자인 시스템 발행. **Default 토글**은 새 프로젝트 기본값 여부.
- **Design Files** 탭에서 SVG/HTML/UI 키트 폴더 확인 가능.
- 결과: **Spotify의 강렬한 다크 모드 + 그린 액센트가 잘 재현**됨.

### 08:39 ~ 11:46 — 슬라이드덱 생성 + 애니메이션 개선

- **Slidedeck** 메뉴 → 프로젝트명 "Spotify Slides"
- 디자인 시스템 자동 적용 확인.
- **Speaker Notes** 옵션 활성화 (텍스트는 슬라이드에 적게, 발표자 노트로 분리).
- 모델: **Opus 4.7** 그대로 사용.
- 이전 영상의 젠스파크 클로우 인트로 스크립트 + "PPT 슬라이드로 만들어 달라" → **10장 + 발표자 노트** 자동 생성.
- 슬라이드 퀄리티가 깔끔하지만 발표자 평가는 "다소 밋밋함" → 추가 프롬프트로 한 단계 발전:

```
전체적인 내용은 유지하되 아래 가이드에 따라 애니메이션 슬라이드로 개선해 줘.

- 배경: WebGL 쉐이더
- 폰트: {지정}
- 컬러: 화이트 + 액센트 1색
- 텍스트 가독성 유지
- 슬라이드별 맥락에 맞는 추상 도형/상징적 그래픽
- 애니메이션: JSX 애니메이션 컴포넌트
```

- 결과:
  - 타임라인에 **플레이 버튼** 생성
  - 배경의 **쉐이더 연기 효과**
  - 슬라이드별 추상 도형 + 상징 그래픽 애니메이션 (예: 시계 그래픽)
  - **텍스트 스트리밍 애니메이션**으로 텍스트가 작성되는 효과
  - **반투명 UI**, 인포그래픽 흐름 표현, 깔끔한 테이블

> 발표자 평가: 프롬프트 두 번 요청으로 이 정도 결과 — "정말 놀랍다"

### 11:47 ~ 13:31 — 4가지 수정 수단

영상이 시연한 수정 방법:

#### Tweaks (커스텀 슬라이더)

- 캔버스 도구의 **Tweaks** 클릭 → 패널 표시
- "적절한 수정 옵션을 만들어 달라"고 프롬프트 → Claude가 **컬러 / 지속 시간 / 강도** 슬라이더를 자동 생성
- 우측 상단 Tweaks 토글 활성화 → 우측 하단 패널에서 실시간 조작
- 시연: 액센트 컬러 변경, 신 지속 시간 조정, 배경 쉐이더 강도 조정 (연기 효과가 밝아졌다 어두워짐), 신 번호 가리기/보이기

#### Drawing (영역 마킹)

- 캔버스 위 영역을 **드로잉**으로 직접 마킹
- 수정 요청: "박스 안에 텍스트가 다 들어가도록"
- **하나의 샘플만 수정 요청 → 같은 패턴이 적용된 세 가지 요소 모두 수정**됨

#### Edit (요소 직접 편집)

- 수정하고 싶은 섹션 선택 → 폰트 / 사이즈 / 웨이트 / 컬러 / 얼라인 등 패널에서 직접 조정
- 시연: 박스의 패딩, 카드의 radius 값 변경 → 모서리가 둥글게 변경됨

#### Presentation Mode (보너스)

- 우측 상단 **Present** → **In Steps** → 발표 모드 진입
- 슬라이드 전체 화면 + **별도 발표자 노트** 패널
- 발표자가 노트를 보면서 더 편안하게 발표 가능

### 13:32 ~ 17:42 — Figma 디자인 시스템 임포트 (★ 발표자 강력 추천)

> **발표자 평가**: "저는 개인적으로 이게 이번 클로드 디자인에서 **가장 핵심적인 기능**이라고 생각합니다." (자막 14:09)

#### 왜 핵심인가?

- Figma Community에는 **이미 글로벌 유명 기업이 직접 만든 퀄리티 높은 디자인 시스템 파일**이 다수 공개되어 있음.
- 이 파일들을 그대로 임포트하면, **별도로 디자인 가이드를 만들 필요 없이** 즉시 완성도 높은 디자인 시스템 베이스를 확보할 수 있음.

#### 시연: IBM Carbon Design System으로 Threads 클론

1. Figma Community에서 **"Carbon Design System"** 검색 (IBM 오픈소스 — 방대한 컴포넌트 라이브러리, 강력한 접근성 기준, 디자인 + 프런트엔드 코드 문서화 — "교과서 같은 디자인 시스템")
2. 좌측 상단 **메뉴 → 파일 → 로컬 복사본 저장** → `.fig` 파일 다운로드
3. Claude Design → **Design Systems → Create**
4. Company Name 입력 + Figma 파일 업로드
5. **Continue to Generation → Generate**
6. Need Review 진행 (이전 워크플로우와 동일) → **Publish**
7. **New Design Using This System** 클릭
8. **Prototype** 메뉴로 이동:
   - Prototype Name 입력
   - Carbon Design System 선택
   - **Wireframe vs Hi-Fidelity** 중 **Hi-Fidelity** 선택
9. 프롬프트: "쓰레드 같은 글 위주로 포스팅하고 공유하는 플랫폼에 주요 키스크린을 만들어 달라"
10. **Clarifying Questions** 표시 — 컨텍스트 부족분에 대한 선택형 질문지
    - 발표자 선택: "Carbon Design System 규칙을 엄격하게 따르도록"
11. 결과:
    - **5개 키스크린**: Home Feed / Thread Detail / Composer / Profile / Discovery
    - **Light Mode + Dark Mode** 두 버전 동시 생성
    - "엔터프라이즈 쓰레드 플랫폼" 컨셉, AI/디자인 시스템 관련 더미 텍스트
    - Carbon Design System을 **그대로 잘 따르고 있음** 확인
12. **Tweaks 패널**에서 추가 조정 가능:
    - 모드 전환 / 라인 헤이트 / 쓰레드 커넥터 / 액센트 컬러
13. 더미 데이터 보강 요청: "이미지 포함해서 그럴싸하게 채워 달라"
    - 결과: **이미지 자동 생성**되어 채워지고, 텍스트도 AI 기술 관련 내용으로 변경됨

#### 발표자 결론

> "이제는 디자인 시스템 하나만 구축해 두면 그것들을 조합해서 디자인을 만들어 주는 것까지 AI에게 위임을 할 수가 있을 것 같습니다." (자막 17:42)

### 17:43 ~ 18:53 — Claude Code 핸드오프 + 추가 결과물

- 완성된 디자인은 **Export → Claude Code 핸드오프**로 개발 작업으로 이어 갈 수 있음.
- 추가 사례 1: **3D 글로브 여행지 웹 프로덕트**
  - 첫 시도(프롬프트만): 어느 정도 구현되지만 퀄리티는 평범
  - GitHub의 **react-globe.gl** 컴포넌트 코드를 복사해 첨부 → **빠르게 퀄리티 향상**
  - 학습: **외부 컴포넌트 코드를 첨부하면 결과물 퀄리티가 크게 올라간다**
- 추가 사례 2: **카드 뉴스** (Prompt Templates)
  - 카드 뉴스에 최적화된 **디자인 가이드 문서를 먼저 자동 생성** (거의 파운데이션 문서급)
  - 이를 바탕으로 카드 뉴스 생성 → 즉시 사용 가능한 퀄리티
  - 이미지/영상도 삽입 가능 (별도 다룸 예고)

## 영상 핵심 인사이트 5가지

1. **디자인 프로세스 패러다임 전환**이 진짜 메시지 — Jenny Wen 인용이 핵심.
2. Claude Design의 진짜 핵심은 **결과물이 아니라 디자인 시스템 자동 생성**.
3. **Figma Community 자산 임포트**가 가장 강력한 활용 패턴.
4. **Tweaks**(Claude가 슬라이더 자체를 디자인) + **Drawing**은 다른 도구에 없는 차별 기능.
5. **비용 부담**은 무시할 수 없으며, 실무 도입 시 ROI 검토가 필요하다.
