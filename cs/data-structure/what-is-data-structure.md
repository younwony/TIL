## 왜 알아야 하는가?

자료구조는 프로그래밍의 기초이자 핵심입니다. 같은 문제라도 어떤 자료구조를 선택하느냐에 따라 성능이 천차만별입니다. 코딩 테스트에서 자료구조 선택이 문제 해결의 핵심이며, 실무에서도 적절한 자료구조 선택이 시스템 성능을 좌우합니다.

## 핵심 개념

- **자료구조**: 데이터를 조직화하고 저장하는 방식
- **추상 자료형 (ADT)**: 자료와 연산을 정의한 명세 (구현 방법 X)
- **시간 복잡도**: 연산에 걸리는 시간 (Big-O)
- **공간 복잡도**: 필요한 메모리 양

## 쉽게 이해하기

**자료구조**를 정리함에 비유할 수 있습니다.

책을 정리하는 방법:
- **배열**: 책꽂이에 순서대로 꽂기 (빠른 접근, 삽입 어려움)
- **연결 리스트**: 책마다 다음 책 위치 메모 (삽입 쉬움, 찾기 어려움)
- **해시 테이블**: 책 제목 첫 글자로 섹션 분류 (빠른 검색)
- **트리**: 책을 카테고리별로 분류 (계층적 정리)

## 상세 설명

### 자료구조의 분류

```
자료구조
├── 선형 자료구조 (Linear)
│   ├── 배열 (Array)
│   ├── 연결 리스트 (Linked List)
│   ├── 스택 (Stack)
│   └── 큐 (Queue)
│
└── 비선형 자료구조 (Non-Linear)
    ├── 트리 (Tree)
    │   ├── 이진 트리
    │   ├── 힙
    │   └── 트라이
    ├── 그래프 (Graph)
    └── 해시 테이블 (Hash Table)
```

### 주요 자료구조 비교

| 자료구조 | 접근 | 검색 | 삽입 | 삭제 | 특징 |
|---------|------|------|------|------|------|
| 배열 | O(1) | O(n) | O(n) | O(n) | 연속 메모리, 인덱스 접근 |
| 연결 리스트 | O(n) | O(n) | O(1) | O(1) | 동적 크기, 포인터 |
| 스택 | O(n) | O(n) | O(1) | O(1) | LIFO |
| 큐 | O(n) | O(n) | O(1) | O(1) | FIFO |
| 해시 테이블 | - | O(1)* | O(1)* | O(1)* | 키-값, 충돌 시 O(n) |
| 이진 탐색 트리 | - | O(log n)* | O(log n)* | O(log n)* | 정렬된 트리, 최악 O(n) |
| 힙 | - | O(n) | O(log n) | O(log n) | 최대/최소값 O(1) |

*평균 시간 복잡도

### 추상 자료형 (ADT) vs 자료구조

```
ADT (무엇을 할 것인가?)        자료구조 (어떻게 구현할 것인가?)
       │                              │
       ▼                              ▼
┌─────────────┐               ┌──────────────────┐
│    리스트    │               │ 배열로 구현       │
│  - 삽입     │    ─────►     │ 연결 리스트로 구현 │
│  - 삭제     │               │                  │
│  - 검색     │               └──────────────────┘
└─────────────┘

예: 스택 ADT
- push(item): 항목 추가
- pop(): 항목 제거
- peek(): 맨 위 확인
- isEmpty(): 비었는지 확인

구현: 배열 or 연결 리스트
```

### 자료구조 선택 기준

```
┌─────────────────────────────────────────────────────────────┐
│                 자료구조 선택 가이드                          │
├─────────────────────────────────────────────────────────────┤
│                                                              │
│  빠른 접근 (인덱스) 필요?                                     │
│      └─ Yes ──► 배열                                         │
│      └─ No ──► 다음 질문으로                                  │
│                                                              │
│  삽입/삭제가 빈번?                                            │
│      └─ Yes ──► 연결 리스트                                   │
│      └─ No ──► 다음 질문으로                                  │
│                                                              │
│  키로 빠른 검색 필요?                                         │
│      └─ Yes ──► 해시 테이블                                   │
│      └─ No ──► 다음 질문으로                                  │
│                                                              │
│  계층 구조/정렬 필요?                                         │
│      └─ Yes ──► 트리 (BST, AVL)                              │
│      └─ No ──► 다음 질문으로                                  │
│                                                              │
│  최대/최소값 빠르게 필요?                                     │
│      └─ Yes ──► 힙                                           │
│      └─ No ──► 다음 질문으로                                  │
│                                                              │
│  관계 표현 필요?                                              │
│      └─ Yes ──► 그래프                                       │
│                                                              │
│  LIFO/FIFO 필요?                                             │
│      └─ LIFO ──► 스택                                        │
│      └─ FIFO ──► 큐                                          │
│                                                              │
└─────────────────────────────────────────────────────────────┘
```

### 실무 사용 예시

| 상황 | 적합한 자료구조 | 이유 |
|------|---------------|------|
| 브라우저 뒤로 가기 | 스택 | LIFO - 가장 최근 페이지 |
| 프린터 대기열 | 큐 | FIFO - 먼저 요청한 순서 |
| 사용자 캐시 | 해시 테이블 | 빠른 조회 (O(1)) |
| 파일 시스템 | 트리 | 계층적 구조 |
| SNS 친구 관계 | 그래프 | 다대다 관계 |
| 작업 스케줄링 | 힙 (우선순위 큐) | 우선순위 기반 처리 |

## 면접 예상 질문

### Q: 배열과 연결 리스트의 차이점은?

A: **배열**은 연속된 메모리에 저장되어 인덱스로 O(1) 접근이 가능하지만, 크기가 고정되고 중간 삽입/삭제가 O(n)입니다. **연결 리스트**는 노드가 포인터로 연결되어 동적 크기와 O(1) 삽입/삭제가 가능하지만, 순차 접근만 가능해 검색이 O(n)입니다. **선택 기준**: 읽기 위주면 배열, 삽입/삭제 위주면 연결 리스트.

### Q: 해시 테이블의 시간 복잡도가 O(1)인 이유는?

A: 해시 함수로 키를 인덱스로 변환하여 배열에 직접 접근하기 때문입니다. **단, 충돌**이 발생하면 최악 O(n)이 될 수 있습니다. 충돌 해결 방법으로 체이닝(연결 리스트)과 오픈 어드레싱(다른 슬롯 탐색)이 있습니다.
