# 자동화란 (What is Automation)

## 왜 알아야 하는가?

개발자의 시간은 제한적입니다. 반복 작업을 자동화하면 더 가치 있는 일에 집중할 수 있습니다. 테스트 실행, 배포, 모니터링 등 많은 작업이 자동화 대상입니다. 현대 DevOps와 CI/CD는 자동화를 기반으로 합니다.

## 핵심 개념

- **스크립트 자동화**: 코드로 작업 자동화 (Bash, Python)
- **RPA**: 사람의 UI 조작을 모방하는 자동화
- **워크플로우 자동화**: 여러 단계를 연결한 자동화
- **CI/CD**: 빌드, 테스트, 배포의 자동화

## 쉽게 이해하기

**자동화**를 공장의 컨베이어 벨트에 비유할 수 있습니다.

```
수동 작업 (사람이 직접):
┌───────┐    ┌───────┐    ┌───────┐
│ 빌드  │ →→ │ 테스트 │ →→ │ 배포  │
│ (수동)│    │ (수동) │    │ (수동)│
└───────┘    └───────┘    └───────┘
   ↑            ↑            ↑
 개발자        개발자        개발자

자동화 (시스템이 자동으로):
┌─────────────────────────────────────┐
│        컨베이어 벨트 (자동화)         │
│  빌드 → 테스트 → 배포                │
└─────────────────────────────────────┘
                ↑
            트리거 (git push)
```

### 스크립트 vs RPA

| 구분 | 스크립트 자동화 | RPA |
|------|----------------|-----|
| 대상 | API, CLI, 파일 | GUI, 화면 조작 |
| 속도 | 빠름 | 상대적 느림 |
| 안정성 | 높음 | UI 변경에 취약 |
| 대상 시스템 | API 제공 시스템 | 레거시, API 없는 시스템 |
| 기술 요구 | 코딩 필요 | 노코드/로우코드 |

### 자동화 대상 선정

```
자동화 우선순위 매트릭스:

      자동화 용이성
           ↑
    높음   │  ★ 즉시 자동화    │  검토 필요
           │  (ROI 높음)       │  (기술 투자)
           ├──────────────────┼─────────────────
           │  선택적 자동화    │  자동화 보류
    낮음   │  (여유 있을 때)   │  (수동 유지)
           └──────────────────┴─────────────────→
                 낮음                높음
                          반복 빈도

★ 우선 자동화 대상:
  - 매일 반복되는 작업
  - 실수 가능성 높은 작업
  - 시간 소모가 큰 작업
  - 표준화된 프로세스
```

### 자동화 사례

#### 1. 개발 자동화

```bash
# 예: Git Hook으로 커밋 전 자동 린트
# .git/hooks/pre-commit
#!/bin/sh
npm run lint
if [ $? -ne 0 ]; then
    echo "❌ Lint 실패. 커밋 취소."
    exit 1
fi
```

#### 2. 배포 자동화

```yaml
# GitHub Actions 예시
name: Deploy
on:
  push:
    branches: [main]
jobs:
  deploy:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - run: npm install
      - run: npm run build
      - run: npm run deploy
```

#### 3. 데이터 처리 자동화

```python
# 매일 자정 실행되는 데이터 정리 스크립트
import schedule
import time

def cleanup_old_logs():
    # 30일 이상 된 로그 삭제
    pass

schedule.every().day.at("00:00").do(cleanup_old_logs)

while True:
    schedule.run_pending()
    time.sleep(60)
```

## 면접 예상 질문

### Q: 자동화 대상을 선정하는 기준은?

A: (1) **반복 빈도**: 매일/매주 반복되는 작업 우선 (2) **소요 시간**: 수동으로 10분 이상 걸리는 작업 (3) **오류 가능성**: 실수하기 쉬운 작업 (4) **표준화 정도**: 프로세스가 명확한 작업. **계산**: ROI = (수동 시간 × 빈도) - 자동화 구축/유지 비용. 단, 과도한 자동화는 오히려 복잡성 증가.

### Q: 스크립트 자동화와 RPA의 차이는?

A: **스크립트**: API/CLI 기반, 빠르고 안정적, 코딩 필요. **RPA**: GUI 조작 기반, UI 변경에 취약, 노코드 가능. **선택 기준**: API 있으면 스크립트, 레거시 시스템이면 RPA. **하이브리드**: API + RPA 조합으로 커버리지 극대화.
