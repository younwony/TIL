# 보안이란 (What is Security)

> `[1] 정의` · 선수 지식: 없음

> 시스템, 데이터, 통신을 위협으로부터 보호하는 기술과 프로세스의 총칭

## 왜 알아야 하는가?

보안은 모든 시스템의 필수 요소입니다. 개발자는 보안을 고려하여 코드를 작성해야 하며, 보안 취약점은 비즈니스에 막대한 피해를 줍니다. 면접에서도 보안 기초 지식을 자주 질문합니다.

## 핵심 개념

- **CIA Triad**: 기밀성, 무결성, 가용성
- **인증 vs 인가**: 신원 확인 vs 권한 확인
- **위협 vs 취약점**: 잠재적 위험 vs 실제 약점
- **심층 방어**: 여러 계층의 보안

## 쉽게 이해하기

**보안**을 집 보안에 비유할 수 있습니다.

- **기밀성**: 도둑이 금고를 열지 못하게 (암호화)
- **무결성**: 도둑이 물건을 바꿔치기 하지 못하게 (해시)
- **가용성**: 주인이 언제든 집에 들어갈 수 있게 (백업)
- **인증**: 초인종 누르면 신분 확인 (로그인)
- **인가**: 손님은 거실만, 가족은 전체 출입 (권한)

## 상세 설명

### CIA Triad (보안의 3요소)

```
              Confidentiality
                 (기밀성)
                    △
                   / \
                  /   \
                 /     \
                /       \
               /    ●    \
              /           \
             /             \
            /_______________ \
    Integrity              Availability
    (무결성)               (가용성)
```

| 요소 | 의미 | 위협 예시 | 대응 |
|------|------|---------|------|
| 기밀성 | 인가된 자만 접근 | 데이터 유출 | 암호화 |
| 무결성 | 데이터 변조 방지 | 데이터 위조 | 해시/서명 |
| 가용성 | 필요 시 접근 가능 | DDoS 공격 | 이중화/백업 |

### 인증 vs 인가

```
사용자 ──► [인증] "당신은 누구?" ──► [인가] "당신은 뭘 할 수 있나?"
           │                         │
           └─ 로그인, MFA            └─ 권한 확인, RBAC
```

| 구분 | 인증 (Authentication) | 인가 (Authorization) |
|------|---------------------|---------------------|
| 질문 | 당신은 누구인가? | 무엇을 할 수 있는가? |
| 검증 | 신원 (Identity) | 권한 (Permission) |
| 방법 | 비밀번호, MFA, 생체인증 | RBAC, ABAC, ACL |
| 시점 | 먼저 | 인증 후 |

### 위협, 취약점, 공격

```
위협 (Threat)          취약점 (Vulnerability)      공격 (Attack)
"도둑이 있을 수 있다"   "창문 잠금이 약하다"        "도둑이 창문을 열었다"
      │                       │                        │
      └──────────────────────┼───────────────────────┘
                              ▼
                          위험 (Risk)
                    "도둑이 창문으로 침입할 확률"
```

### 보안 위협 유형

| 유형 | 설명 | 예시 |
|------|------|------|
| 스푸핑 | 신원 위장 | IP/DNS 스푸핑 |
| 변조 | 데이터 수정 | Man-in-the-Middle |
| 부인 | 행위 부정 | 거래 부인 |
| 정보 노출 | 기밀 유출 | 데이터 유출 |
| 서비스 거부 | 가용성 방해 | DDoS |
| 권한 상승 | 불법 권한 획득 | 루트 권한 탈취 |

### 심층 방어 (Defense in Depth)

```
┌─────────────────────────────────────────────────────────┐
│                    물리적 보안                           │
│  ┌───────────────────────────────────────────────────┐ │
│  │                 네트워크 보안                       │ │
│  │  ┌─────────────────────────────────────────────┐  │ │
│  │  │               호스트 보안                     │  │ │
│  │  │  ┌───────────────────────────────────────┐  │  │ │
│  │  │  │            애플리케이션 보안            │  │  │ │
│  │  │  │  ┌─────────────────────────────────┐  │  │  │ │
│  │  │  │  │           데이터 보안            │  │  │  │ │
│  │  │  │  │         (암호화, 백업)           │  │  │  │ │
│  │  │  │  └─────────────────────────────────┘  │  │  │ │
│  │  │  │    (인증, 인가, 입력 검증)            │  │  │ │
│  │  │  └───────────────────────────────────────┘  │  │ │
│  │  │      (패치, 안티바이러스, 설정)              │  │ │
│  │  └─────────────────────────────────────────────┘  │ │
│  │       (방화벽, IDS/IPS, VPN)                       │ │
│  └───────────────────────────────────────────────────┘ │
│          (출입 통제, CCTV)                              │
└─────────────────────────────────────────────────────────┘
```

### 보안 원칙

| 원칙 | 설명 |
|------|------|
| 최소 권한 | 필요한 최소한의 권한만 부여 |
| 기본 거부 | 명시적 허용 외 모두 거부 |
| 심층 방어 | 여러 계층에서 보호 |
| 실패 안전 | 실패 시 안전한 상태로 |
| 분리 | 권한/시스템 분리 |

## 트레이드오프

| 항목 | 트레이드오프 |
|------|-------------|
| 보안 vs 편의성 | 강한 보안 ↔ 사용자 불편 |
| 보안 vs 비용 | 높은 보안 ↔ 높은 비용 |
| 보안 vs 성능 | 암호화 ↔ 처리 시간 |

## 면접 예상 질문

### Q: CIA Triad란 무엇인가요?

A: 보안의 3대 요소입니다. **기밀성(Confidentiality)**: 인가된 사용자만 정보 접근 (암호화). **무결성(Integrity)**: 정보가 변조되지 않음 보장 (해시, 서명). **가용성(Availability)**: 필요할 때 정보 접근 가능 (이중화, 백업). 세 요소의 균형이 중요합니다.

### Q: 인증과 인가의 차이는?

A: **인증**: "당신이 누구인가?" - 신원 확인 (비밀번호, MFA). **인가**: "당신이 무엇을 할 수 있는가?" - 권한 확인 (RBAC). **순서**: 항상 인증이 먼저, 인가가 나중. **예시**: 로그인(인증) 후 관리자 메뉴 접근 가능 여부 확인(인가).
