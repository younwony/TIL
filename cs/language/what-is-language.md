# 프로그래밍 언어란 (What is Programming Language)

> `[1] 정의` · 선수 지식: 없음

> 사람이 컴퓨터에게 명령을 전달하기 위한 형식화된 언어 체계

`#프로그래밍언어` `#ProgrammingLanguage` `#컴파일` `#Compile` `#인터프리터` `#Interpreter` `#고급언어` `#HighLevelLanguage` `#저급언어` `#LowLevelLanguage` `#정적타입` `#StaticType` `#동적타입` `#DynamicType` `#강타입` `#StrongType` `#약타입` `#WeakType` `#컴파일러` `#Compiler` `#런타임` `#Runtime` `#바이트코드` `#Bytecode` `#JIT` `#AOT` `#패러다임`

## 왜 알아야 하는가?

프로그래밍 언어의 특성을 이해하면 적절한 언어를 선택하고, 언어별 장단점을 활용할 수 있습니다. 컴파일/인터프리터 차이, 타입 시스템 이해는 버그 방지와 성능 최적화에 직결됩니다.

## 핵심 개념

- **고급 언어 vs 저급 언어**: 추상화 수준의 차이
- **컴파일 vs 인터프리터**: 실행 방식의 차이
- **정적 타입 vs 동적 타입**: 타입 검사 시점의 차이
- **강타입 vs 약타입**: 타입 변환 엄격성의 차이

## 쉽게 이해하기

**프로그래밍 언어**를 외국어에 비유할 수 있습니다.

- **고급 언어**: 영어 (사람이 이해하기 쉬움)
- **저급 언어**: 기계 신호 (컴퓨터가 직접 이해)
- **컴파일러**: 번역가 (책 전체를 미리 번역)
- **인터프리터**: 통역사 (한 문장씩 실시간 통역)

## 상세 설명

### 언어 수준 분류

```
추상화 수준
    ↑
    │   Python, Java, Kotlin ← 고급 언어 (사람 친화적)
    │
    │   C, C++ ← 중급 언어
    │
    │   Assembly ← 저급 언어
    │
    ↓   Machine Code (0, 1) ← 기계어 (컴퓨터 직접 실행)
```

### 컴파일 vs 인터프리터

| 방식 | 동작 | 장점 | 단점 | 예시 |
|------|------|------|------|------|
| 컴파일 | 전체 → 기계어 → 실행 | 빠른 실행 | 느린 개발 | C, Go, Rust |
| 인터프리터 | 한 줄씩 → 실행 | 빠른 개발 | 느린 실행 | Python, Ruby |
| 하이브리드 | 컴파일 + JIT | 균형 | 복잡도 | Java, Kotlin |

```
컴파일 언어 (C):
┌────────┐     ┌──────────┐     ┌────────┐
│ .c 파일 │ ──► │ 컴파일러  │ ──► │ .exe   │ ──► 실행
└────────┘     └──────────┘     └────────┘
              (한 번만)         (매우 빠름)

인터프리터 언어 (Python):
┌────────┐     ┌────────────┐
│ .py 파일│ ──► │ 인터프리터 │ ──► 실행
└────────┘     └────────────┘
              (매번, 느림)

하이브리드 (Java):
┌────────┐     ┌──────────┐     ┌────────┐     ┌─────┐
│ .java  │ ──► │ javac    │ ──► │ .class │ ──► │ JVM │ ──► 실행
└────────┘     └──────────┘     └────────┘     └─────┘
              (컴파일)         (바이트코드)    (JIT)
```

### 타입 시스템

#### 정적 타입 vs 동적 타입

```java
// 정적 타입 (Java) - 컴파일 시 타입 검사
String name = "John";
name = 123;  // 컴파일 에러!
```

```python
# 동적 타입 (Python) - 런타임 시 타입 검사
name = "John"
name = 123   # OK (런타임에 타입 변경)
```

| 구분 | 정적 타입 | 동적 타입 |
|------|----------|----------|
| 타입 검사 시점 | 컴파일 | 런타임 |
| 타입 오류 발견 | 빠름 (개발 중) | 느림 (실행 중) |
| 코드 양 | 많음 | 적음 |
| IDE 지원 | 강력함 | 제한적 |
| 예시 | Java, Kotlin, Go | Python, JavaScript |

#### 강타입 vs 약타입

```python
# 강타입 (Python)
"3" + 5   # TypeError! 자동 변환 안 함
```

```javascript
// 약타입 (JavaScript)
"3" + 5   // "35" (암시적 변환)
```

### 타입 추론 (Type Inference)

정적 타입이면서 타입 명시 생략:

```java
// Java 10+
var name = "John";  // String으로 추론
var count = 10;     // int로 추론
```

```kotlin
// Kotlin
val name = "John"   // String으로 추론
val count = 10      // Int로 추론
```

### JIT vs AOT

```
JIT (Just-In-Time):
┌────────────┐     ┌─────────────┐     ┌──────────┐
│ 바이트코드  │ ──► │ 실행 중 분석 │ ──► │ 기계어   │
└────────────┘     │ (핫스팟 찾기)│     │ (캐싱)   │
                   └─────────────┘     └──────────┘
장점: 런타임 최적화 가능
단점: 시작 시간 느림 (웜업 필요)
예: Java, JavaScript

AOT (Ahead-Of-Time):
┌────────────┐     ┌──────────┐
│ 소스코드    │ ──► │ 기계어   │ ──► 즉시 실행
└────────────┘     └──────────┘
장점: 빠른 시작
단점: 런타임 최적화 불가
예: C, Go, GraalVM Native Image
```

### 언어 선택 가이드

| 상황 | 권장 언어 | 이유 |
|------|----------|------|
| 시스템/임베디드 | C, Rust | 메모리 제어, 성능 |
| 웹 백엔드 | Java, Go, Python | 생태계, 생산성 |
| 모바일 (Android) | Kotlin | 공식 지원, 간결 |
| 데이터 분석 | Python | 라이브러리, 간결 |
| 스크립팅 | Python, Shell | 빠른 개발 |

## 트레이드오프

| 항목 | 트레이드오프 |
|------|-------------|
| 성능 vs 생산성 | C(성능) ↔ Python(생산성) |
| 안전성 vs 유연성 | 정적 타입(안전) ↔ 동적 타입(유연) |
| 제어력 vs 편의성 | 저수준(제어) ↔ 고수준(편의) |

## 면접 예상 질문

### Q: Java가 컴파일 언어인가요, 인터프리터 언어인가요?

A: **하이브리드**입니다. (1) javac로 소스를 **바이트코드**(.class)로 컴파일 (2) JVM이 바이트코드를 인터프리터로 실행 (3) 자주 실행되는 코드는 **JIT 컴파일러**가 기계어로 변환하여 캐싱. **장점**: "Write Once, Run Anywhere" + 런타임 최적화.

### Q: 정적 타입과 동적 타입의 장단점은?

A: **정적 타입**: 컴파일 시 타입 오류 발견, IDE 자동완성 강력, 리팩토링 안전. **동적 타입**: 빠른 프로토타이핑, 코드 간결, 유연한 메타프로그래밍. **선택 기준**: 대규모 프로젝트/팀 → 정적 타입, 빠른 개발/스크립팅 → 동적 타입.
