# CS 학습 문서 작성 가이드

이 문서는 CS 지식 문서 작성 시 Claude가 참고하는 가이드입니다.

## 카테고리 및 주제

### Network (네트워크)

| 주제 | 설명 |
|------|------|
| OSI 7계층 | 물리, 데이터링크, 네트워크, 전송, 세션, 표현, 응용 계층 |
| TCP/IP | TCP vs UDP, 3-way handshake, 4-way handshake |
| HTTP | HTTP/1.1, HTTP/2, HTTP/3, HTTPS, 상태코드 |
| DNS | 도메인 네임 시스템, 동작 원리, 레코드 종류 |
| 로드밸런싱 | L4/L7, 알고리즘, 헬스체크 |
| CDN | 콘텐츠 전송 네트워크, 캐싱 전략 |
| 웹소켓 | 실시간 통신, HTTP와 차이점 |
| REST API | RESTful 설계, 멱등성, HATEOAS |
| gRPC | Protocol Buffers, HTTP/2 기반 RPC |
| 쿠키/세션 | 상태 관리, 보안 고려사항 |

### OS (운영체제)

| 주제 | 설명 |
|------|------|
| 프로세스 | 프로세스 vs 스레드, PCB, 컨텍스트 스위칭 |

| 동기화 | 뮤텍스, 세마포어, 모니터, 데드락 |
| 메모리 관리 | 가상 메모리, 페이징, 세그멘테이션 |
| 파일 시스템 | inode, 저널링, 파일 할당 방식 |
| I/O | 동기/비동기, 블로킹/논블로킹 |
| 캐시 | CPU 캐시, 캐시 지역성, 캐시 일관성 |
| 인터럽트 | 하드웨어/소프트웨어 인터럽트, 인터럽트 핸들링 |

### DB (데이터베이스)

### System Design (시스템 설계)

## 작성 규칙

### 파일명

- kebab-case 사용: `tcp-handshake.md`, `process-vs-thread.md`
- 영문 소문자: `index.md` (O), `Index.md` (X)

### 내용 작성

| 항목 | 규칙 |
|------|------|
| 핵심 개념 | 3-5개의 불릿포인트로 요약 |
| 쉽게 이해하기 | 실생활 비유, 일상 예시로 개념 설명 (비전공자 대상) |
| 상세 설명 | 개념별 소제목(H3)으로 구분 |
| 예제 코드 | 언어 명시, 주석으로 설명 추가 |
| 면접 질문 | 실제 면접에서 자주 나오는 질문 2-3개 |
| 참고 자료 | 공식 문서, 신뢰할 수 있는 출처 |

### 품질 기준

- **정확성**: 공식 문서 및 신뢰할 수 있는 출처 기반
- **이해 용이성**: 비전공자도 이해할 수 있는 실생활 비유와 예시 포함
- **간결성**: 불필요한 내용 배제, 핵심에 집중
- **실용성**: 면접 대비, 실무 적용 가능한 내용
- **일관성**: 동일한 템플릿과 스타일 유지

## "쉽게 이해하기" 작성 가이드

비전공자도 이해할 수 있도록 **실생활 비유**와 **일상 예시**를 활용합니다.

### 좋은 비유의 조건

| 조건 | 설명 |
|------|------|
| 친숙함 | 누구나 아는 일상적인 상황 사용 |
| 정확성 | 개념의 핵심 특성을 잘 반영 |
| 단순함 | 복잡한 비유보다 간단한 비유 선호 |

### 비유 예시

| 개념 | 비유 |
|------|------|
| TCP 3-way handshake | 전화 통화: "여보세요?" → "네, 들려요" → "좋아요, 통화 시작합니다" |
| 프로세스 vs 스레드 | 프로세스 = 공장, 스레드 = 공장 안의 작업자들 (자원 공유) |
| 캐시 | 자주 쓰는 물건을 책상 위에 두는 것 (빠른 접근) |
| 데드락 | 좁은 복도에서 서로 비켜주기를 기다리며 둘 다 못 지나가는 상황 |
| 인덱스 | 책의 목차/색인 (전체를 안 읽고 원하는 페이지로 바로 이동) |
| 가상 메모리 | 도서관 대출: 전체 책을 소유하지 않아도 필요할 때 빌려서 사용 |
| 로드밸런싱 | 마트 계산대: 줄이 짧은 곳으로 고객 안내 |
| 트랜잭션 | 은행 송금: 출금과 입금이 둘 다 성공하거나 둘 다 취소되어야 함 |
| 해시 | 사물함 번호: 이름으로 번호를 계산해서 바로 찾아감 |
| 스택/큐 | 스택 = 접시 쌓기 (LIFO), 큐 = 줄서기 (FIFO) |

### 작성 형식

```markdown
## 쉽게 이해하기

**{핵심 개념}**을 {비유 대상}에 비유할 수 있습니다.

{비유 설명 - 2~3문장}

예를 들어, {구체적인 상황 예시}
```

### 작성 예시: 페이징

```markdown
## 쉽게 이해하기

**페이징**을 책 대여 시스템에 비유할 수 있습니다.

도서관에서 책을 빌릴 때, 책장 전체를 가져오지 않고 필요한 책만 빌려옵니다.
컴퓨터도 마찬가지로, 프로그램 전체를 메모리에 올리지 않고 필요한 부분(페이지)만 올려서 사용합니다.

예를 들어, 100페이지짜리 문서를 편집할 때 1~10페이지만 보고 있다면,
나머지 90페이지는 디스크에 두고 현재 보는 부분만 메모리에 로드합니다.
다른 페이지로 이동하면 그때 해당 페이지를 가져옵니다 (페이지 폴트).
```
