# CS 학습 문서 작성 가이드

이 문서는 CS 지식 문서 작성 시 Claude가 참고하는 가이드입니다.

## 카테고리 및 주제

### Network (네트워크)

| 주제 | 설명 |
|------|------|
| OSI 7계층 | 물리, 데이터링크, 네트워크, 전송, 세션, 표현, 응용 계층 |
| TCP/IP | TCP vs UDP, 3-way handshake, 4-way handshake |
| HTTP | HTTP/1.1, HTTP/2, HTTP/3, HTTPS, 상태코드 |
| DNS | 도메인 네임 시스템, 동작 원리, 레코드 종류 |
| 로드밸런싱 | L4/L7, 알고리즘, 헬스체크 |
| CDN | 콘텐츠 전송 네트워크, 캐싱 전략 |
| 웹소켓 | 실시간 통신, HTTP와 차이점 |
| REST API | RESTful 설계, 멱등성, HATEOAS |
| gRPC | Protocol Buffers, HTTP/2 기반 RPC |
| 쿠키/세션 | 상태 관리, 보안 고려사항 |

### OS (운영체제)

| 주제 | 설명 |
|------|------|
| 프로세스 | 프로세스 vs 스레드, PCB, 컨텍스트 스위칭 |
| 스케줄링 | FCFS, SJF, RR, 우선순위, 멀티레벨 큐 |
| 동기화 | 뮤텍스, 세마포어, 모니터, 데드락 |
| 메모리 관리 | 가상 메모리, 페이징, 세그멘테이션 |
| 파일 시스템 | inode, 저널링, 파일 할당 방식 |
| I/O | 동기/비동기, 블로킹/논블로킹 |
| 캐시 | CPU 캐시, 캐시 지역성, 캐시 일관성 |
| 인터럽트 | 하드웨어/소프트웨어 인터럽트, 인터럽트 핸들링 |

### DB (데이터베이스)

| 주제 | 설명 |
|------|------|
| 인덱스 | B-Tree, B+Tree, Hash, 클러스터드/논클러스터드 |
| 트랜잭션 | ACID, 격리 수준, 락 |
| 정규화 | 1NF~5NF, 반정규화 |
| SQL | JOIN, 서브쿼리, 실행계획 |
| NoSQL | MongoDB, Redis, Cassandra, 특징과 선택 기준 |
| 레플리케이션 | 마스터-슬레이브, 마스터-마스터 |
| 샤딩 | 수평 분할, 샤드 키 선택 |
| 커넥션 풀 | HikariCP, 풀 사이징 |
| MVCC | 다중 버전 동시성 제어 |

### Algorithm (알고리즘)

| 주제 | 설명 |
|------|------|
| 시간복잡도 | Big-O, 최선/평균/최악 |
| 정렬 | 퀵, 병합, 힙, 기수, 계수 정렬 |
| 탐색 | 이진 탐색, DFS, BFS |
| 그래프 | 다익스트라, 벨만포드, 플로이드워셜, MST |
| DP | 메모이제이션, 타뷸레이션, 대표 문제들 |
| 그리디 | 탐욕 알고리즘, 적용 조건 |
| 분할정복 | 병합정렬, 퀵정렬, 이진탐색 |
| 자료구조 | 스택, 큐, 힙, 트리, 해시, 그래프 |

### System Design (시스템 설계)

| 주제 | 설명 |
|------|------|
| 확장성 | 수직/수평 확장, 스케일 아웃 전략 |
| 가용성 | HA, 페일오버, 재해복구 |
| 캐싱 전략 | 캐시 무효화, Write-through/back/around |
| 메시지 큐 | Kafka, RabbitMQ, 이벤트 드리븐 |
| 마이크로서비스 | MSA vs 모놀리식, 서비스 분리 |
| API 게이트웨이 | 라우팅, 인증, Rate limiting |
| CAP 정리 | 일관성, 가용성, 분할 내성 |
| 분산 시스템 | 합의 알고리즘, 분산 락 |

## 작성 규칙

### 파일명

- kebab-case 사용: `tcp-handshake.md`, `process-vs-thread.md`
- 영문 소문자: `index.md` (O), `Index.md` (X)

### 내용 작성

| 항목 | 규칙 |
|------|------|
| 핵심 개념 | 3-5개의 불릿포인트로 요약 |
| 상세 설명 | 개념별 소제목(H3)으로 구분 |
| 예제 코드 | 언어 명시, 주석으로 설명 추가 |
| 면접 질문 | 실제 면접에서 자주 나오는 질문 2-3개 |
| 참고 자료 | 공식 문서, 신뢰할 수 있는 출처 |

### 품질 기준

- **정확성**: 공식 문서 및 신뢰할 수 있는 출처 기반
- **간결성**: 불필요한 내용 배제, 핵심에 집중
- **실용성**: 면접 대비, 실무 적용 가능한 내용
- **일관성**: 동일한 템플릿과 스타일 유지
