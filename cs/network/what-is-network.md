## 왜 알아야 하는가?

모든 웹 서비스, API, 마이크로서비스는 네트워크 위에서 동작합니다. 네트워크의 기본 원리를 이해해야 성능 문제(지연, 대역폭), 보안 문제(방화벽, TLS), 장애 분석(패킷 손실, DNS 오류)을 해결할 수 있습니다. 백엔드 개발자에게 네트워크 지식은 필수입니다.

## 핵심 개념

- **네트워크**: 컴퓨터들이 데이터를 주고받을 수 있게 연결된 시스템
- **프로토콜**: 통신 규칙 (HTTP, TCP/IP 등)
- **패킷**: 네트워크에서 전송되는 데이터 조각
- **IP 주소**: 네트워크에서 기기를 식별하는 주소
- **대역폭/지연시간**: 네트워크 성능의 두 축

## 쉽게 이해하기

**네트워크**를 택배 시스템에 비유할 수 있습니다.

- **IP 주소 = 집 주소**: 어디로 보낼지 식별
- **패킷 = 택배 상자**: 데이터를 담은 단위
- **라우터 = 물류센터**: 어디로 보낼지 결정
- **프로토콜 = 배송 규칙**: 포장 방법, 취급 주의 등
- **대역폭 = 도로 폭**: 한 번에 얼마나 많이 보낼 수 있는지
- **지연시간 = 배송 시간**: 도착까지 걸리는 시간

### 네트워크 분류

#### 규모에 따른 분류

| 유형 | 범위 | 예시 |
|------|------|------|
| **PAN** | 개인 영역 (~10m) | 블루투스, USB |
| **LAN** | 로컬 영역 (~1km) | 사무실, 가정 내부 |
| **MAN** | 도시 영역 (~50km) | 도시 내 네트워크 |
| **WAN** | 광역 (국가/세계) | 인터넷 |

#### 토폴로지 (네트워크 구조)

```
버스 (Bus)                  스타 (Star)              링 (Ring)
───┬───┬───┬───           ┌───┐                   ┌───┐
   │   │   │              │ A │                   │ A │
 ┌─┴─┐┌┴┐┌─┴─┐          ┌─┴───┴─┐               ┌─┴───┴─┐
 │ A ││B││ C │          │ 허브  │               │       │
 └───┘└─┘└───┘          └┬─┬─┬─┘             ┌─┴─┐   ┌─┴─┐
                        B  C  D              │ D │───│ B │
                                             └─┬─┘   └─┬─┘
                                               └───────┘
                                                   C
```

| 토폴로지 | 장점 | 단점 |
|---------|------|------|
| 버스 | 간단, 저렴 | 단일 장애점 |
| 스타 | 관리 용이, 장애 격리 | 허브 장애 시 전체 마비 |
| 링 | 균등한 접근 | 하나 고장 시 전체 영향 |
| 메시 | 고가용성 | 복잡, 비용 높음 |

### 핵심 용어

#### IP 주소 (Internet Protocol Address)

**역할**: 네트워크에서 기기를 식별하는 주소

```
IPv4: 192.168.1.1     (32비트, 약 43억 개)
IPv6: 2001:0db8:85a3:0000:0000:8a2e:0370:7334 (128비트)

특수 IP:
- 127.0.0.1 (localhost): 자기 자신
- 192.168.x.x: 사설 IP (내부망)
- 0.0.0.0: 모든 네트워크 인터페이스
```

#### 포트 (Port)

**역할**: 같은 IP에서 여러 서비스를 구분

```
IP 주소 = 건물 주소
포트 = 호실 번호

예: 192.168.1.1:80
    └─ IP ─┘ └포트

주요 포트:
- 80: HTTP
- 443: HTTPS
- 22: SSH
- 3306: MySQL
- 5432: PostgreSQL
```

#### 패킷 (Packet)

**역할**: 네트워크에서 전송되는 데이터 단위

```
┌─────────────────────────────────────────────┐
│                   패킷                       │
├─────────────┬───────────────────────────────┤
│   헤더      │          페이로드              │
│ (송신/수신  │       (실제 데이터)            │
│  주소 등)   │                               │
└─────────────┴───────────────────────────────┘

큰 데이터 → 여러 패킷으로 분할 → 전송 → 수신 측에서 재조립
```

### 네트워크 장비

| 장비 | 역할 | OSI 계층 |
|------|------|---------|
| **허브 (Hub)** | 모든 포트로 데이터 전송 | 1계층 (물리) |
| **스위치 (Switch)** | MAC 주소로 목적지만 전송 | 2계층 (데이터 링크) |
| **라우터 (Router)** | IP 주소로 최적 경로 결정 | 3계층 (네트워크) |
| **방화벽 (Firewall)** | 트래픽 필터링, 보안 | 3~7계층 |

### 네트워크 성능 지표

| 지표 | 설명 | 비유 |
|------|------|------|
| **대역폭 (Bandwidth)** | 초당 전송 가능한 데이터량 | 도로의 차선 수 |
| **지연시간 (Latency)** | 데이터가 도착하는 데 걸리는 시간 | 목적지까지 거리 |
| **처리량 (Throughput)** | 실제로 전송되는 데이터량 | 실제 주행 속도 |
| **패킷 손실 (Packet Loss)** | 전송 중 손실된 패킷 비율 | 분실된 택배 |

```
대역폭이 높아도 지연시간이 길 수 있음!
예: 위성 인터넷 - 대역폭 높음, 지연시간 500ms+
```

## 트레이드오프

| 항목 | 대역폭 ↑ | 지연시간 ↓ |
|------|---------|-----------|
| 비용 | 높음 | 높음 |
| 중요한 상황 | 대용량 파일 전송 | 실시간 게임, 화상회의 |
| 개선 방법 | 회선 증설 | 물리적 거리 단축, CDN |

## 면접 예상 질문

### Q: IP 주소와 포트의 역할은?

A: **IP 주소**는 네트워크에서 기기를 식별하는 주소입니다. **포트**는 같은 IP에서 여러 서비스를 구분합니다. **비유**: IP 주소는 건물 주소, 포트는 호실 번호입니다. 예를 들어, 웹 서버는 80번 포트, SSH는 22번 포트를 사용합니다. **왜 필요한가?** 한 서버에서 여러 서비스(웹, DB, SSH)를 동시에 운영하려면 포트로 구분해야 합니다.

### Q: 대역폭과 지연시간의 차이는?

A: **대역폭**은 단위 시간당 전송 가능한 데이터량(Mbps)이고, **지연시간**은 데이터가 도착하는 데 걸리는 시간(ms)입니다. **비유**: 대역폭은 도로의 차선 수, 지연시간은 목적지까지 거리입니다. **예시**: 대역폭이 높아도 위성 인터넷은 지연시간이 500ms 이상입니다. 실시간 게임은 지연시간이 중요하고, 대용량 파일 전송은 대역폭이 중요합니다.

### Q: 라우터와 스위치의 차이는?

A: **스위치**는 MAC 주소를 기반으로 같은 네트워크(LAN) 내에서 패킷을 전달합니다. **라우터**는 IP 주소를 기반으로 서로 다른 네트워크 간 패킷을 전달하고 최적 경로를 결정합니다. **왜 둘 다 필요한가?** 스위치는 사무실 내부 통신, 라우터는 인터넷(외부)과의 통신을 담당합니다.
