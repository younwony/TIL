## 왜 알아야 하는가?

프로그래밍은 소프트웨어 개발의 핵심 활동입니다. 프로그래밍의 본질과 다양한 패러다임을 이해하면 문제에 맞는 적절한 접근 방식을 선택할 수 있습니다. 모든 개발 학습의 출발점입니다.

## 핵심 개념

- **프로그래밍**: 문제를 해결하는 절차를 코드로 표현
- **프로그래밍 언어**: 사람과 컴퓨터 간의 의사소통 도구
- **패러다임**: 프로그래밍의 사고 방식과 스타일
- **추상화**: 복잡함을 단순화하여 다루는 것

## 쉽게 이해하기

**프로그래밍**을 요리 레시피 작성에 비유할 수 있습니다.

- **재료** = 데이터 (변수)
- **조리 도구** = 연산자, 함수
- **조리 순서** = 알고리즘
- **레시피 언어** = 프로그래밍 언어

레시피대로 따라하면 누구나 같은 요리를 만들 수 있듯이, 프로그램대로 실행하면 컴퓨터가 같은 결과를 냅니다.

### 패러다임 비교: 리스트 합계

**절차지향**:
```c
int sum = 0;
for (int i = 0; i < n; i++) {
    sum += numbers[i];
}
```

**객체지향**:
```java
class Calculator {
    int sum(List<Integer> numbers) {
        int total = 0;
        for (int num : numbers) {
            total += num;
        }
        return total;
    }
}
```

**함수형**:
```java
int sum = numbers.stream()
    .reduce(0, Integer::sum);
```

### 컴파일 vs 인터프리터

```
컴파일 (Compile):
┌──────────┐     ┌──────────┐     ┌──────────┐
│ 소스코드  │ ──► │ 컴파일러  │ ──► │ 실행파일  │
│  .java   │     │          │     │  .class  │
└──────────┘     └──────────┘     └──────────┘
예: Java, C, Go

인터프리터 (Interpreter):
┌──────────┐     ┌──────────────┐
│ 소스코드  │ ──► │  인터프리터   │ ──► 실행
│  .py     │     │  (한 줄씩)   │
└──────────┘     └──────────────┘
예: Python, JavaScript, Ruby
```

| 방식 | 장점 | 단점 |
|------|------|------|
| 컴파일 | 빠른 실행, 최적화 | 느린 개발 사이클 |
| 인터프리터 | 빠른 개발, 유연성 | 느린 실행 |

### 좋은 프로그래밍의 원칙

1. **가독성**: 코드는 사람이 읽기 쉬워야 함
2. **단순성**: 필요한 것만, 복잡도 최소화
3. **재사용성**: 중복을 피하고 모듈화
4. **유지보수성**: 변경이 쉬운 구조

## 면접 예상 질문

### Q: 객체지향과 함수형 프로그래밍의 차이는?

A: **객체지향**은 데이터와 행동을 객체로 묶어 상태를 관리하고, **함수형**은 불변 데이터와 순수 함수로 부작용을 최소화합니다. **선택 기준**: 상태 관리가 복잡하면 객체지향, 데이터 변환이 주요 작업이면 함수형이 적합합니다. 현대 언어는 두 패러다임을 혼합 지원합니다.

### Q: 컴파일 언어와 인터프리터 언어의 차이는?

A: **컴파일 언어**는 전체 코드를 기계어로 변환 후 실행하여 빠르지만 개발 사이클이 길고, **인터프리터 언어**는 한 줄씩 실행하여 개발이 빠르지만 실행이 느립니다. Java는 컴파일(bytecode)+인터프리터(JVM)를 결합한 하이브리드 방식입니다.
