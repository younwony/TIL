# Part 1: 프로의 마음가짐

## 핵심 내용

### 책임과 의무를 가져라

프로 개발자는 자신의 코드와 결정에 대해 책임을 진다.

### 무엇보다도 해를 끼치지 마라

#### 기능에 해를 끼치지 마라
- 오류를 만들지 않도록 최선을 다한다
- QA가 오류를 찾지 못해야 한다

#### 구조에 해를 끼치지 마라
- 소프트웨어는 변경하기 쉬워야 한다
- 코드를 변경할 때마다 더 좋은 구조로 만든다

### 직업 윤리

**100% 테스트 커버리지는 선택이 아니라 필수다**

### 프로가 알아야 할 최소한의 기술 목록

| 분야 | 내용 |
|------|------|
| 디자인 패턴 | GOF 24개 패턴, POSA 패턴 |
| 설계 원칙 | SOLID 원칙, 컴포넌트 원칙 |
| 방법론 | XP, 스크럼, 린, 칸반, 폭포수, 구조적 분석, 구조적 설계 |
| 실천 원칙 | TDD, 객체지향 설계, 구조적 프로그래밍, CI, 짝 프로그래밍 |
| 분석 도구 | UML, DFD, 구조 차트, 페트리 넷, 상태 전이도, 흐름 차트, 결정 테이블 |

### 지속적 학습

- 끊임없이 배우고 연습한다
- 새로운 언어와 기술을 익힌다
- 자신의 분야 외의 것도 학습한다

### 연습

- 코딩 도장(kata)을 통해 실력을 유지한다
- 실제 업무 외의 시간에 연습한다

### 함께 일하기

- 다른 개발자와 함께 프로그래밍한다
- 짝 프로그래밍과 코드 리뷰를 실천한다

### 멘토링

- 후배 개발자를 가르치는 것이 배우는 최선의 방법이다

### 업무 도메인 이해

- 자신이 코딩하는 업무 도메인을 깊이 이해한다
- 도메인 전문가와 소통한다

### 고용주/고객과 동일시

- 고용주의 문제가 자신의 문제라고 생각한다
- 고객의 관점에서 생각한다

### 겸손

- 프로는 자신의 능력에 자부심을 갖지만, 오만하지 않다
- 실수를 인정하고 그로부터 배운다
