# Part 4: 코딩

## 핵심 내용

### 준비된 자세

코딩할 때 다음을 보장해야 한다:
1. **코드는 반드시 동작해야 한다**
2. **고객의 문제를 해결해야 한다**
3. **기존 시스템과 잘 통합되어야 한다**
4. **다른 프로그래머가 읽기 쉬워야 한다**

### 새벽 3시에 짠 코드

- 지쳤을 때 코드를 작성하지 마라
- 피로한 상태의 코드는 나중에 더 큰 비용을 초래한다

### 근심이 담긴 코드

- 개인적인 걱정이 있을 때는 코딩을 피한다
- 집중하지 못하면 버그가 생긴다
- 문제를 해결하거나, 걱정을 잠시 접어두는 연습이 필요하다

### 몰입 영역 (The Zone)

#### 몰입 회피
- "몰입" 상태가 항상 좋은 것은 아니다
- 전체적인 관점을 잃기 쉽다

#### 페어 프로그래밍
- 짝 프로그래밍은 몰입을 건강하게 유지한다
- 상대방이 큰 그림을 봐준다

#### 음악
- 어떤 개발자에게는 도움이 되고, 어떤 개발자에게는 방해가 된다

#### 외부 방해
- 방해는 피할 수 없다
- 정중하게 대응하고 빠르게 다시 집중한다

### 진퇴양난에 빠진 글쟁이 (Writer's Block)

막혔을 때의 해결법:
- 짝 프로그래밍을 한다
- 잠시 다른 일을 한다
- 산책을 한다

### 창의적인 입력

- 다양한 입력이 창의력을 높인다
- 책, 영화, 음악 등 다양한 경험이 필요하다

### 디버깅

- TDD로 디버깅 시간을 대폭 줄일 수 있다
- 디버깅 시간도 코딩 시간에 포함된다

### 속도 조절

- 코딩은 마라톤이다
- 지속 가능한 페이스를 유지한다

### 언제 걸어 나가야 할지 알기

- 피로하면 휴식을 취한다
- 막히면 잠시 떠난다
- 샤워나 운전 중에 해결책이 떠오르기도 한다

### 일정을 못 지키다

**3가지 완료일자를 제시하라:**
1. 최선의 경우
2. 평균적인 경우
3. 최악의 경우

### 희망

- 희망은 프로젝트 킬러다
- 문제가 있으면 빨리 알려라
- 실패에 대비한 후퇴 계획을 세워라

### 질주

- 부추김에 넘어가지 마라
- 초과 근무가 해결책이 아니다
- 속도를 높이려다 품질을 떨어뜨리지 마라

### 초과 근무

- 지속적인 초과근무는 실패의 징조다
- 2주 이상의 초과근무에는 동의하지 마라

### 가짜 출시

- "완료"했다고 거짓말하는 것은 최악의 프로답지 못한 행동이다

### 완료 정의

- "완료"는 자동화된 인수 테스트를 통과한 것이다
- 명확한 완료 기준을 정의한다

### 도움

#### 다른 사람 돕기
- 동료와 함께 코드를 작성한다
- 도움 요청에 응한다

#### 도움 받기
- 도움이 필요하면 요청한다
- 도움에 감사한다

#### 멘토링
- 경험 많은 개발자는 후배를 멘토링할 의무가 있다
