# Part 12: 함께 일하기 (Collaboration)

## 핵심 내용

### 프로그래머 vs 보통 사람들

- 프로그래머는 종종 사람들과 어울리는 것보다 컴퓨터와 일하는 것을 선호한다
- 하지만 프로 개발자는 팀의 일원으로서 협력해야 한다

### 고용주

- 고용주의 문제가 자신의 문제라고 생각한다
- 비즈니스 목표를 이해하고 그에 맞게 일한다
- 사업이 성공해야 개발자도 성공한다

### 팀

#### 코드 소유권
- 코드는 개인의 것이 아니라 팀의 것이다
- 모든 코드를 함께 소유한다
- 누구나 어떤 코드든 수정할 수 있다

#### 짝 프로그래밍
- 지식을 공유하는 최고의 방법이다
- 코드 품질이 높아진다
- 팀의 결속력이 강화된다

#### 코드 리뷰
- 모든 코드는 리뷰되어야 한다
- 버그를 조기에 발견한다
- 지식을 공유한다

### 소뇌 (협력의 물리적 측면)

- 같은 공간에서 일하면 의사소통이 원활하다
- 비언어적 의사소통이 가능하다
- 즉각적인 피드백이 가능하다

### 협력 방법

#### 1. 함께 앉기
- 물리적으로 가까이 앉는다
- 의사소통 비용을 줄인다

#### 2. 함께 프로그래밍하기
```
짝 프로그래밍:
- 한 명이 타이핑 (드라이버)
- 한 명이 검토 (네비게이터)
- 역할을 자주 교체
```

#### 3. 지식 공유
- 문서화
- 코드 리뷰
- 기술 세미나
- 페어 프로그래밍

## 결론

- 프로그래밍은 팀 스포츠다
- 혼자 일하는 것보다 함께 일하는 것이 더 효과적이다
- 코드는 팀 전체의 책임이다
- 적극적으로 협력한다
