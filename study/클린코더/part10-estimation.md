# Part 10: 추정 (Estimation)

## 핵심 내용

### 추정이란 무엇인가?

- 추정은 **약속이 아니다**
- 추정은 **확률 분포**다
- 비즈니스는 약속을 원하고, 개발자는 추정을 제공한다

### 추정 vs 약속

| 구분 | 추정 | 약속 |
|------|------|------|
| 성격 | 어림짐작 | 확정적 책임 |
| 변경 | 가능 | 불가 |
| 확률 | 범위로 표현 | 단일 값 |

### PERT (Program Evaluation and Review Technique)

3점 추정 기법:
- **O (Optimistic)**: 최선의 경우
- **N (Nominal)**: 일반적인 경우
- **P (Pessimistic)**: 최악의 경우

```
기대값 = (O + 4N + P) / 6
표준편차 = (P - O) / 6
```

**예시:**
```
O = 1일, N = 3일, P = 12일

기대값 = (1 + 12 + 12) / 6 = 4.2일
표준편차 = (12 - 1) / 6 = 1.8일

예상 완료: 4.2일 ± 1.8일 (2.4일 ~ 6일)
```

### 업무 추정하기

#### 와이드밴드 델파이
1. 팀이 모여서 각자 추정한다
2. 추정치를 공개하고 논의한다
3. 합의에 도달할 때까지 반복한다

#### 플래닝 포커
- 각자 카드로 추정치를 제시
- 차이가 크면 논의
- 합의될 때까지 반복

### 큰 수의 법칙

- 큰 업무는 작은 업무로 분할한다
- 작은 업무의 추정 오차는 서로 상쇄된다
- 전체 추정의 정확도가 높아진다

```
큰 업무 1개 추정: 불확실
작은 업무 10개 추정 합: 더 정확
```

## 결론

- 추정은 불확실성을 포함한다
- 3점 추정으로 범위를 표현한다
- 작은 업무로 분할하여 정확도를 높인다
- 추정을 약속으로 오해하지 않도록 한다
