# Part 13: 팀과 프로젝트 (Teams and Projects)

## 핵심 내용

### 갈아서 만들었나요? (Gelled Teams)

**Gelled Team(결속된 팀)의 특징:**
- 함께 일하는 방법을 알고 있다
- 서로의 강점과 약점을 안다
- 신뢰가 구축되어 있다
- 의사소통이 원활하다

### 팀 형성 비용

- 팀이 결속되기까지 시간이 걸린다
- 새 프로젝트마다 팀을 해체하고 재구성하면 비용이 크다
- 팀을 유지하고 프로젝트를 팀에 할당하는 것이 효율적이다

### 프로젝트 vs 팀

**전통적 방식:**
```
프로젝트 A → 임시 팀 A 구성 → 프로젝트 완료 → 팀 해체
프로젝트 B → 임시 팀 B 구성 → 프로젝트 완료 → 팀 해체
```

**권장 방식:**
```
결속된 팀 → 프로젝트 A → 프로젝트 B → 프로젝트 C → ...
```

### 팀의 구성

이상적인 팀 구성:
- 프로그래머: 7~8명
- 테스터: 2명
- 분석가: 2명
- 프로젝트 관리자: 1명

### 팀의 속도

- 팀은 자신들의 속도를 알고 있다
- 예측 가능한 결과를 낸다
- 경영진은 이를 신뢰할 수 있다

### 프로젝트 오너십

**프로젝트별 팀 할당 문제:**
- 팀이 특정 프로젝트에 얽매인다
- 프로젝트가 끝나면 팀이 해체된다
- 축적된 팀워크가 사라진다

**팀별 프로젝트 할당:**
- 팀은 유지된다
- 프로젝트가 팀에 할당된다
- 팀의 전문성이 축적된다

## 결론

- 결속된 팀을 형성하는 데는 시간이 걸린다
- 팀을 해체하지 말고 유지한다
- 프로젝트를 팀에 할당한다
- 팀의 속도와 능력을 신뢰한다
