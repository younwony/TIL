# Part 2: 아니라고 말하기

## 핵심 내용

### 반대하는 역할

- 프로는 권위에 맞서야 할 때가 있다
- "아니오"라고 말하는 것도 전문가의 역할이다

### 손익관계가 높을 때

- 중요한 결정일수록 "아니오"라고 말할 용기가 필요하다
- 불가능한 일정에 동의하면 결국 모두가 손해를 본다

### 팀 플레이어

- 진정한 팀 플레이어는 무조건 "예"라고 말하는 사람이 아니다
- 팀에 도움이 되는 정직한 의견을 말하는 것이 팀 플레이다

### "예"라고 말하는 비용

- 불가능한 약속의 대가는 크다
- 지킬 수 없는 약속은 신뢰를 무너뜨린다

### 훌륭한 코드는 불가능한가?

- 시간이 충분하지 않다는 핑계로 나쁜 코드를 작성하지 마라
- 빠르게 가는 유일한 방법은 잘 가는 것이다

### 코드 임파서블

- 불가능한 요구에는 거절해야 한다
- 전문가로서의 판단을 포기하면 안 된다

## 어떻게 "아니오"라고 말할 것인가

### 대안 제시
```
"그 일정에는 불가능하지만, X일까지는 가능합니다."
"전체 기능은 어렵지만, 핵심 기능만 먼저 완성할 수 있습니다."
```

### 이유 설명
- 왜 불가능한지 구체적으로 설명한다
- 기술적 제약과 현실을 솔직히 공유한다

### 협상
- 범위, 일정, 품질 중 조정 가능한 것을 제안한다
- 비즈니스와 기술의 균형점을 찾는다
