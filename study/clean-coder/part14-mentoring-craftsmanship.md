# Part 14: 스승과 제자 그리고 장인 정신 (Mentoring and Craftsmanship)

## 핵심 내용

### 실패의 정도

- 소프트웨어 산업은 체계적인 교육 시스템이 부족하다
- 많은 개발자가 제대로 된 멘토링 없이 성장한다
- 이로 인해 기술과 윤리가 제대로 전수되지 않는다

### 스승과 제자

#### 전통적인 도제 시스템
1. **견습공 (Apprentice)**
   - 경험 많은 개발자 밑에서 배운다
   - 코드 리뷰를 받는다
   - 점진적으로 책임을 맡는다

2. **숙련공 (Journeyman)**
   - 자율적으로 일할 수 있다
   - 아직 완전한 전문가는 아니다
   - 여러 마스터 밑에서 배운다

3. **장인 (Master)**
   - 높은 수준의 기술과 경험
   - 다른 개발자를 멘토링한다
   - 기술과 윤리를 전수한다

### 수습기간

**신입 개발자 수습 과정:**
1. 첫 1년은 멘토와 함께 일한다
2. 점진적으로 책임을 늘린다
3. 코드 리뷰와 피드백을 받는다
4. 기술과 문화를 익힌다

### 장인 정신 (Craftsmanship)

#### 장인의 특징
- 품질에 대한 자부심
- 지속적인 학습
- 기술의 전수
- 윤리적 행동

#### 장인 정신의 가치
```
- 동작하는 소프트웨어뿐만 아니라, 잘 만들어진 소프트웨어
- 변화에 대응하는 것뿐만 아니라, 꾸준히 가치를 더하는 것
- 개인과의 상호작용뿐만 아니라, 전문가들의 커뮤니티
- 고객과의 협력뿐만 아니라, 생산적인 파트너십
```

### 멘토링 방법

#### 1. 짝 프로그래밍
- 실시간으로 지식 전수
- 즉각적인 피드백

#### 2. 코드 리뷰
- 코드 품질 향상
- 기술과 스타일 전수

#### 3. 기술 세미나
- 팀 내 지식 공유
- 새로운 기술 소개

#### 4. 독서 모임
- 함께 책을 읽고 토론
- 다양한 관점 학습

## 결론

- 멘토링은 프로 개발자의 의무다
- 후배 개발자를 성장시키는 것이 팀과 산업 전체에 기여하는 것이다
- 장인 정신은 단순한 기술이 아니라 태도와 가치관이다
- 끊임없이 배우고, 배운 것을 전수한다
