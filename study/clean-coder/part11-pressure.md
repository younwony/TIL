# Part 11: 압박 (Pressure)

## 핵심 내용

### 압박 피하기

#### 약속 피하기
- 지킬 수 없는 약속은 하지 않는다
- 불합리한 일정에는 "아니오"라고 말한다

#### 깨끗하게 유지하기
- 압박 상황에서도 좋은 코드를 작성한다
- "나중에 정리하겠다"는 거짓말이다
- 빠르게 가는 방법은 잘 가는 것이다

#### 규율 지키기
- TDD를 버리지 않는다
- 코드 리뷰를 건너뛰지 않는다
- 테스트를 줄이지 않는다

### 압박 다루기

#### 침착하게
- 패닉에 빠지지 않는다
- 깊게 호흡하고 문제를 분석한다

#### 소통하기
- 문제가 있으면 빨리 알린다
- 혼자 끙끙대지 않는다
- 도움을 요청한다

#### 규율에 의지하기
- 평소에 하던 대로 한다
- 위기 상황에서 규율이 무너지면 규율이 없는 것이다
- 평소에 TDD를 하지 않으면서 위기에 한다는 것은 거짓말이다

#### 도움 받기
- 짝 프로그래밍을 한다
- 팀원에게 도움을 요청한다
- 혼자보다 둘이 더 빠르다

### 위기와 규율

```
위기 상황에서도 평소와 같은 방식으로 일한다면,
그것이 당신이 진정으로 믿는 방식이다.

위기 상황에서 규율을 버린다면,
평소에도 그 규율을 진정으로 믿지 않는 것이다.
```

## 결론

- 압박은 피할 수 없지만 관리할 수 있다
- 불합리한 약속으로 압박을 자초하지 않는다
- 압박 상황에서도 규율을 지킨다
- 소통하고 도움을 구한다
