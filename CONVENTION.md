## 폴더 구조

### 최상위 구조

```
TIL/
├── README.md              # 저장소 소개
├── CONVENTION.md          # 컨벤션 가이드 (이 문서)
├── cs/                    # CS 지식 (마크다운 중심)
└── study/                 # 스터디 (프로젝트 단위)
```

### CS 폴더 구조

```
cs/
├── {category}/            # 카테고리명 (kebab-case)
│   ├── README.md          # 카테고리 소개 및 목차
│   └── {topic}.md         # 개별 주제 문서
```

### Study 폴더 구조

```
study/
├── {study-name}/          # 스터디명 (kebab-case)
│   ├── README.md          # 스터디 소개 및 목차
│   └── {chapter}/         # 챕터/주차별 폴더
│       └── {item}.md      # 개별 아이템 문서
```

## 네이밍 규칙

### 폴더명

| 규칙 | 예시 |
|------|------|
| **kebab-case** 사용 (소문자, 하이픈 구분) | `effective-java`, `java-live`, `clean-coder` |
| **영문** 사용 (한글 폴더명 지양) | `clean-coder` (O), `클린코더` (X) |
| **단수형** 사용 | `algorithm` (O), `algorithms` (X) |

### 파일명

| 타입 | 규칙 | 예시 |
|------|------|------|
| 마크다운 | kebab-case | `item01.md`, `part01-professionalism.md` |
| Java | PascalCase (클래스명) | `NutritionFacts.java` |
| 알고리즘 | `a{문제번호}_{문제명}.java` | `a1000_더하기.java` |

### 챕터/파트 번호

| 타입 | 형식 | 예시 |
|------|------|------|
| 챕터 | `chapter{00}` | `chapter02`, `chapter10` |
| 주차 | `week{00}` | `week04`, `week12` |
| 파트 | `part{00}` | `part01`, `part14` |
| 아이템 | `item{00}` | `item01`, `item25` |

## 마크다운 작성 규칙

### README.md 템플릿

```markdown
# {Study Name} Study

{한 줄 설명}

> 원본 레포지토리: {URL} (있는 경우)

## 프로젝트 정보 (있는 경우)

- **언어:** {언어}
- **빌드 도구:** {빌드 도구}
- **프레임워크:** {프레임워크}

## 목차

### {Chapter/Week/Part 제목}
- [{아이템 제목}]({상대경로})
- ...

## 참고

{추가 참고 사항}
```

### 개별 문서 템플릿

```markdown
# {제목}

## 핵심 정리

{핵심 내용 요약}

## 본문

{상세 내용}

### 소제목

{소제목 내용}

## 예제 코드

```java
// 코드 예제
```

## 참고

- 원본 코드: {링크}
- 관련 문서: {링크}
```

### 마크다운 스타일

| 요소 | 규칙 |
|------|------|
| 제목 | H1은 파일당 1개, H2부터 계층 구조 |
| 코드 블록 | 언어 명시 (```java, ```bash 등) |
| 링크 | 상대 경로 사용 |
| 표 | 헤더 구분선 필수 |
| 리스트 | 하이픈(`-`) 사용 |
| 강조 | `**굵게**`, `*기울임*`, `` `코드` `` |

## 코드 파일 규칙

### Java

| 규칙 | 설명 |
|------|------|
| 클래스명 | PascalCase |
| 메서드/변수명 | camelCase |
| 상수명 | UPPER_SNAKE_CASE |
| 패키지명 | 소문자 |

```
study/algorithm/{플랫폼}/{난이도 또는 카테고리}/
```

| 플랫폼 | 폴더명 | 파일명 예시 |
|--------|--------|------------|
| 백준 | `boj/bronze/level1` | `a1000_더하기.java` |
| 프로그래머스 | `programmers/level1` | `타겟넘버.java` |
| LeetCode | `leetcode/easy` | `TwoSum.java` |

## Git 규칙

### 커밋 메시지

```
<type>: <subject>

<body> (선택)
```

### Type 종류

| Type | 설명 | 예시 |
|------|------|------|
| `docs` | 문서 추가/수정 | `docs: Effective Java Item 1 추가` |
| `feat` | 새로운 기능/내용 | `feat: 알고리즘 문제 풀이 추가` |
| `fix` | 오류 수정 | `fix: 오타 수정` |
| `refactor` | 구조 변경 | `refactor: 폴더 구조 정리` |
| `chore` | 기타 작업 | `chore: gitignore 수정` |

### 브랜치

- `main`: 기본 브랜치, 안정된 상태 유지
- 필요시 feature 브랜치 사용: `feature/{기능명}`
