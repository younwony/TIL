# Claude Code 사용량 점검

## /usages 체크 결과
```

● Claude Code 사용량 요약

  현황

  ┌──────┬──────────────┬───────┐
  │ 기간 │  주요 원인   │ 비율  │
  ├──────┼──────────────┼───────┤
  │ 주간 │ 서브에이전트 │ 84%   │
  ├──────┼──────────────┼───────┤
  │ 주간 │ 긴 컨텍스트  │ 45%   │
  ├──────┼──────────────┼───────┤
  │ 일간 │ 서브에이전트 │ 6세션 │
  ├──────┼──────────────┼───────┤
  │ 일간 │ 병렬 세션    │ 2세션 │
  ├──────┼──────────────┼───────┤
  │ 일간 │ 긴 컨텍스트  │ 1세션 │
  └──────┴──────────────┴───────┘

  ---
  핵심 원인 2가지

  1. 서브에이전트 과다 호출 (가장 큰 원인)
  - /work-plan-start, /self-review, /team-review 등 에이전트 집약 스킬 사용 시 발생
  - 에이전트 1개 = API 요청 N번

  2. 컨텍스트 누적
  - 대화가 길어질수록 비용 증가 (캐시 있어도 마찬가지)

  ---
  개선 방법

  ┌──────────┬─────────────────────────────────┬────────┐
  │ 우선순위 │              방법               │  효과  │
  ├──────────┼─────────────────────────────────┼────────┤
  │ ★★★      │ 에이전트 불필요한 호출 줄이기   │ 즉각적 │
  ├──────────┼─────────────────────────────────┼────────┤
  │ ★★★      │ 작업 중간 /compact 습관화       │ 즉각적 │
  ├──────────┼─────────────────────────────────┼────────┤
  │ ★★       │ 단순 에이전트에 Haiku 모델 설정 │ 장기적 │
  ├──────────┼─────────────────────────────────┼────────┤
  │ ★★       │ 새 작업 전 /clear               │ 장기적 │
  └──────────┴─────────────────────────────────┴────────┘

  ▎ 요약하면 "에이전트를 줄이고, 컨텍스트를 자주 정리" 하는 것이 가장 효과적입니다.
```

로컬 + 글로벌 관련 스킬, 에이전트 등을 체크해서 절약가능한 부분 있는지 체크.
