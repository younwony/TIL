# Claude Code 릴리스 노트 한글 정리

Claude Code의 릴리스 노트를 한글로 요약 정리해줘.

## 입력 처리

사용자 인자: `$ARGUMENTS`

인자에 따라 처리 범위를 결정:
- **버전 번호** (예: `2.1.47`, `2.0.0`): 해당 버전의 릴리스 노트만 정리
- **"최근N개"** (예: `최근5개`, `최근10개`): 최근 N개 버전을 정리
- **"전체"** 또는 인자 없음: 전체 릴리스 노트 정리 (주요 버전 위주로 요약)
- **버전 범위** (예: `2.0.0~2.1.0`): 해당 범위의 버전들을 정리

## 릴리스 노트 수집

1. 현재 대화 컨텍스트에 릴리스 노트가 이미 있으면 그것을 사용
2. 없으면 `claude --print "/release-notes"` 명령으로 릴리스 노트를 가져온다

## 정리 규칙

### 한글 번역 원칙
- 기술 용어는 영문을 병기한다 (예: "하위 에이전트 (Sub-agent)")
- 기능명, 명령어, 설정값은 원문 그대로 유지 (예: `/release-notes`, `--print`, `settings.json`)
- 코드, 경로, 환경변수는 번역하지 않는다

### 카테고리 분류

각 버전의 변경사항을 다음 카테고리로 분류한다:

| 카테고리 | 설명 |
|----------|------|
| **새 기능** | 새롭게 추가된 기능 |
| **개선** | 기존 기능의 성능, UX, 안정성 향상 |
| **버그 수정** | 오류 및 문제 해결 |
| **변경** | 동작 방식이 바뀐 사항 (Breaking Change 포함) |
| **플랫폼** | Windows, macOS, Linux 등 특정 플랫폼 관련 |
| **SDK/API** | SDK, API 관련 변경 |
| **IDE** | VSCode 등 IDE 확장 관련 |

### 출력 형식

```markdown
# Claude Code 릴리스 노트 (한글)

> 정리 일시: {현재 날짜}
> 정리 범위: {버전 범위}

## Version {버전}

### 주요 하이라이트
- {이 버전에서 가장 중요한 1~3가지 변경사항을 강조}

### 새 기능
- {새 기능 설명}

### 개선
- {개선 사항}

### 버그 수정
- {수정된 버그}

### 변경
- {변경된 동작}

---
```

### 요약 모드 (전체 정리 시)

전체 릴리스 노트를 정리할 때는 **메이저/마이너 버전 단위**로 그룹핑하여 핵심만 정리:

```markdown
## v2.1.x 시리즈 (주요 변경사항)

### v2.1.47 - {한줄 요약}
- 주요: {핵심 변경 1~2개}

### v2.1.32 - {한줄 요약}
- 주요: {핵심 변경 1~2개}
```

## 저장

정리된 내용을 `cs/tool/claude-code-release-notes.md` 파일로 저장한다.
- 파일이 이미 존재하면 기존 내용과 병합 (새 버전은 상단에 추가)
- 파일이 없으면 새로 생성

## 완료 후

정리된 버전 수와 파일 저장 경로를 안내한다.
