# HANDOFF.md 작성

지금까지의 작업 내용을 HANDOFF.md 파일로 정리해줘.
다음 에이전트가 이 파일만 읽고 작업을 이어갈 수 있도록
시도한 것, 성공한 것, 실패한 것, 다음 단계를 명확히 작성해줘.

## 분석할 내용

1. 현재 대화에서 진행한 **작업 목표**를 파악
2. **완료된 작업**을 체크리스트로 정리 (결과 요약 포함)
3. **실패한 시도**를 원인과 함께 기록 (가장 중요!)
4. **다음 단계**를 우선순위와 함께 나열
5. **관련 파일** 경로와 역할 정리
6. **주의사항** 및 제약사항 기록

## HANDOFF.md 형식

```markdown
# HANDOFF: {작업 제목}

## 작업 목표
{이 작업이 최종적으로 달성해야 하는 것}

## 현재 상태
{전체 진행률과 현재 위치 - 예: "3/5 단계 완료"}

## 완료된 작업
- [x] {완료 항목} - {결과 요약}

## 실패한 시도
- {시도 내용} → 실패 이유: {원인}

## 다음 단계
- [ ] {다음 할 일} - {접근 방법 힌트}

## 관련 파일
- `path/to/file` - {역할 설명}

## 주의사항
- {다음 에이전트가 반드시 알아야 할 제약사항}
```

## 작성 후 안내

HANDOFF.md 작성이 완료되면 다음을 안내해줘:

1. ✅ HANDOFF.md 작성 완료
2. 다음 명령어로 컨텍스트를 리셋하세요: `/clear`
3. 새 세션에서 이렇게 시작하세요: `@HANDOFF.md 이 문서를 읽고 다음 단계부터 이어서 진행해줘`
