# AI Slop Detect

AI가 생성한 코드에서 불필요한 패턴을 찾아내는 스킬이다.
"AI Slop"이란 AI가 습관적으로 만들어내는 그럴듯하지만 실제로는 불필요한 코드 — 과잉 추상화, 미사용 코드, 방어적 프로그래밍 남용 등 — 을 말한다.

## 왜 이 스킬이 필요한가

AI 코드 생성 도구는 "안전하고 완전한" 코드를 만들려는 경향이 있다.
그 결과 한 번만 쓰이는 인터페이스, 발생할 수 없는 예외의 처리, 아무도 읽지 않을 주석 같은 것들이 쌓인다.
이런 코드는 당장은 해가 없어 보이지만, 유지보수 비용을 높이고 코드베이스를 비대하게 만든다.

## AI Slop 체크리스트

분석 시 아래 항목을 순서대로 점검한다. 각 항목에 해당하는 코드를 발견하면 심각도와 함께 기록한다.

### 1. 미사용 코드 (Dead Code)
- 사용되지 않는 import, 변수, 메서드, 클래스
- 호출되지 않는 private 메서드
- 도달할 수 없는 코드 블록 (unreachable code)
- **심각도**: 제거 필수

### 2. 과잉 추상화 (Over-Abstraction)
- 구현체가 1개뿐인 인터페이스
- 한 번만 호출되는 유틸리티/헬퍼 메서드
- 불필요한 팩토리/빌더/전략 패턴 (직접 생성으로 충분한 경우)
- 과도한 상속 계층 (3단계 이상)
- **심각도**: 제거 권장

### 3. YAGNI 위반 (You Aren't Gonna Need It)
- 현재 요구사항에 없는 설정 가능성 (configurable)
- 사용되지 않는 확장 포인트
- "나중에 필요할 수도 있는" 파라미터/옵션
- 빈 구현의 훅/콜백
- **심각도**: 제거 권장

### 4. 방어적 프로그래밍 남용
- 발생 불가능한 시나리오에 대한 null 체크/에러 핸들링
- 프레임워크가 이미 보장하는 것에 대한 중복 검증
- 내부 메서드 호출에 대한 과도한 입력 검증 (시스템 경계가 아닌 곳)
- **심각도**: 제거 권장

### 5. 과도한 주석/문서화
- 자명한 코드에 달린 주석 (`// 리스트를 반환한다` → `return list`)
- getter/setter에 달린 Javadoc
- 코드와 동일한 내용을 반복하는 주석
- **심각도**: 참고

### 6. 구조적 비대
- 3줄이면 될 것을 별도 클래스로 분리
- 단순 데이터 전달에 불필요한 래퍼 객체
- 한 파일에서만 쓰이는 enum/상수 클래스 (인라인으로 충분)
- **심각도**: 참고

## 워크플로우

### Step 1: 변경 파일 식별

```bash
# staged + unstaged + untracked 변경 파일
git diff --cached --name-only
git diff --name-only
git ls-files --others --exclude-standard
```

변경 파일이 없으면 "변경된 파일이 없습니다"로 종료한다.
`.md`, `.yml`, `.json` 등 비코드 파일은 분석 대상에서 제외한다.

### Step 2: 코드 분석

변경된 각 파일을 Read 도구로 읽고, AI Slop 체크리스트 6개 항목을 순서대로 점검한다.

**분석 기준**:
- 변경된 코드뿐 아니라, 변경과 함께 추가된 새 파일도 포함
- 기존 코드가 아닌 **이번 변경에서 추가/수정된 부분**에 집중
- CLAUDE.md의 "기능 구현" 섹션 규칙과 대조 (미사용 기능 제거 원칙)

### Step 3: 결과 보고

발견 사항을 심각도별로 분류하여 보고한다.

```markdown
## AI Slop 분석 결과

### 제거 필수 (Critical)
| # | 파일 | 라인 | 항목 | 설명 |
|---|------|------|------|------|
| 1 | Foo.java | 23 | 미사용 코드 | 호출되지 않는 private 메서드 `bar()` |

### 제거 권장 (Warning)
| # | 파일 | 라인 | 항목 | 설명 |
|---|------|------|------|------|

### 참고 (Info)
| # | 파일 | 라인 | 항목 | 설명 |
|---|------|------|------|------|

### 요약
- 총 {N}건 발견 (필수 {n1} / 권장 {n2} / 참고 {n3})
- AI Slop 점수: {10 - 감점} / 10 (높을수록 깨끗)
```

### Step 4: 수정 제안

"제거 필수" 항목에 대해 자동 수정을 제안한다.
사용자가 승인하면 Edit 도구로 수정을 적용한다.

## 주의사항

- 이 스킬은 **AI가 생성한 코드**에 특화되어 있다. 사람이 의도적으로 작성한 확장 포인트까지 제거하지 않도록 주의한다.
- 프로젝트의 CLAUDE.md에 명시된 규칙(예: `@Data` 금지, `Optional` 필드 사용 금지 등)은 이 스킬이 아닌 `code-refactor` 에이전트가 담당한다.
- 테스트 코드는 분석 대상에서 제외한다 (테스트의 "과잉"은 판단이 다르다).
