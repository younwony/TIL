# Out-of-Scope 지식 베이스

거절된 enhancement를 보관하는 폴더. **같은 enhancement가 재제안될 때 빠르게 거절할 근거**를 제공.

## 위치

`.out-of-scope/{이슈번호}-{kebab-case-제목}.md`

또는 Track 시스템 사용 시: `.claude/out-of-scope/{이슈번호}-{제목}.md`

## 본문 구조

```markdown
> *이 파일은 triage 중 AI가 생성했습니다.*

# {이슈 제목}

**이슈**: #{N} (link)
**거절 일**: YYYY-MM-DD
**거절 이유**: {한 줄 요약}

## 원래 제안 (Original proposal)

리포터가 제안한 내용. 원본 본문을 그대로 인용하거나 요약.

## 왜 거절했나 (Why rejected)

- 이유 1
- 이유 2

## 만약 이게 다시 제안된다면 (If proposed again)

거절 근거가 여전히 유효한 조건. 또는 어떤 변화가 있어야 재검토할지.

## 관련 항목

- 관련 이슈 / ADR / CONTEXT.md 용어
```

## 사용법

- `/triage` 가 새 enhancement를 평가할 때 **먼저 `.out-of-scope/*.md`를 스캔**해서 유사 거절 사례가 있는지 확인.
- 사례가 있으면 메인테이너에게 surface: "이 제안은 #N에서 이전에 X 이유로 거절되었습니다. 그 이유가 여전히 유효한지 확인이 필요해 보입니다."
- 메인테이너가 "다시 거절"이라고 하면 새 이슈도 wontfix + .out-of-scope/ 갱신.

## 작성 규칙

1. **원본 그대로 보존**. 거절 이유가 미래에 무효화될 수 있으므로 원 제안을 정확히 기록.
2. **거절 이유는 actionable**. "이건 별로다" X. "X 시나리오에서 동작 안 함, Y 대안이 더 우월" O.
3. **재검토 조건 명시**. 거절 이유가 무엇이 바뀌면 무효화되는지.
