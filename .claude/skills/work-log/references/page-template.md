## 작성 팁

- 비개발자가 가장 먼저 보는 섹션은 "한눈에 보기"와 "이 작업은 무엇인가요?" — 여기에 충분한 배경 설명을 넣어야 합니다.
- ASCII 다이어그램은 코드 블록(```) 안에 작성해야 Confluence에서 깨지지 않습니다.
- 관련 없는 섹션은 생략하세요. DB 변경이 없으면 데이터베이스 섹션을 빼는 식입니다.
- 코드 스니펫은 핵심 로직만 발췌하고, 전체 코드를 붙여넣지 마세요.
- Jira 상태(Status)는 자주 변경되므로 문서에 포함하지 마세요.
