# ASCII 다이어그램 가이드

작업 로그 문서에서 처리 흐름, 아키텍처, 데이터 관계를 시각화할 때 사용하는 ASCII 드로잉 참조입니다.

---

## 기본 요소

### 박스

```
┌─────────┐  ╔═════════╗  +----------+
│  일반   │  ║  강조   ║  |  대체    |
└─────────┘  ╚═════════╝  +----------+
```

- `┌─┐└─┘` : 일반 박스. 대부분의 컴포넌트에 사용
- `╔═╗╚═╝` : 강조 박스. 핵심 컴포넌트나 주의가 필요한 영역에 사용
- `+-+` : ASCII 호환 박스. 유니코드 지원이 불확실한 환경의 폴백

### 화살표

```
─▶  (오른쪽)    ◀─  (왼쪽)
│▼  (아래)      ▲│  (위)
───▶  (긴 화살표)
```

### 연결선

```
─────  (가로선)
│     (세로선)
├──   (T 분기 — 가로로 갈라지는 지점)
└──   (L 코너 — 마지막 하위 항목)
┬     (위에서 아래로 분기)
┴     (아래에서 위로 합류)
```

## 자주 쓰는 패턴

### 수직 흐름 (요청-응답)

```
┌─────────────────┐
│    사용자       │
└────────┬────────┘
         │ 1. 요청
         ▼
┌─────────────────┐
│   Controller    │
└────────┬────────┘
         │ 2. 처리
         ▼
┌─────────────────┐
│    Service      │
└────────┬────────┘
         │ 3. 조회
         ▼
┌─────────────────┐
│    Database     │
└─────────────────┘
```

### 수평 파이프라인

```
┌─────────┐    ┌─────────┐    ┌─────────┐
│ Step 1  │───▶│ Step 2  │───▶│ Step 3  │
│{처리1}  │    │{처리2}  │    │{처리3}  │
└─────────┘    └─────────┘    └─────────┘
```

### 레이어 아키텍처

```
┌──────────────────────────────────┐
│         Frontend Layer           │
│  ┌──────────┐  ┌──────────────┐  │
│  │   View   │  │  JavaScript  │  │
│  └──────────┘  └──────────────┘  │
└───────────────┬──────────────────┘
                │
┌───────────────┴──────────────────┐
│        Controller Layer          │
└───────────────┬──────────────────┘
                │
┌───────────────┴──────────────────┐
│         Service Layer            │
└───────────────┬──────────────────┘
                │
┌───────────────┴──────────────────┐
│        Repository Layer          │
└──────────────────────────────────┘
```

### 테이블 관계

```
┌─────────────┐       ┌─────────────┐
│  Parent     │──────▶│  Child      │
│             │  1:N  │             │
└─────────────┘       └─────────────┘
```

### 트리 구조 (폴더, 계층)

```
Root
├── Branch A
│   ├── Leaf A-1
│   └── Leaf A-2
├── Branch B
│   └── Leaf B-1
└── Branch C
```

## 작성 규칙

- 다이어그램은 코드 블록(```) 안에 작성합니다. Confluence에서 깨지지 않게 하려면 이 규칙을 지켜야 합니다.
- 박스 너비는 내용에 맞춰 조정하되, 같은 다이어그램 내에서는 너비를 통일하면 가독성이 좋아집니다.
- 화살표 옆에 번호나 설명을 붙여 흐름 순서를 명확히 합니다 (예: `│ 1. 요청 전송`).
- 복잡한 흐름은 한 다이어그램에 모두 넣지 말고 단계별로 분리합니다.
