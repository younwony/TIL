# SVG 직접 생성 스킬

## 왜 SVG 직접 생성인가?

- **정확한 레이아웃 제어**: 픽셀 단위 위치·크기·정렬
- **한글 완벽 지원**: 인코딩 문제 없이 한글 텍스트 삽입
- **도구 설치 불필요**: 브라우저에서 바로 렌더링

## 언제 사용하는가?

| 상황 | 사용 도구 |
|------|----------|
| 흐름도, 시퀀스, 클래스 다이어그램 | **Mermaid** 우선 |
| 계층 구조, 패킷/프레임 구조, 메모리 레이아웃 | **SVG** |
| 네트워크 토폴로지, 아키텍처 구성도 | **SVG** |
| 정확한 위치·크기·정렬이 필수인 경우 | **SVG** |
| 좌우 비교, 상태 다이어그램 | **SVG** |

## SVG 생성 규칙

### 기본 구조

```xml
<svg xmlns="http://www.w3.org/2000/svg" viewBox="0 0 {width} {height}">
  <defs>
    <!-- 그라디언트, 마커 정의 -->
  </defs>
  <!-- 간선 먼저, 노드 나중 -->
</svg>
```

## 작성 절차

1. **유형 결정**: 내용에 맞는 다이어그램 유형 선택 (`references/templates.md` 참조)
2. **SVG 파일 생성**: `cs/{category}/images/{diagram-name}.svg`
3. **마크다운에서 참조**: `![설명](./images/{diagram-name}.svg)`
4. **git add 실행**

## 주의사항

1. **파일명**: kebab-case 사용 (예: `osi-7-layer-structure.svg`)
2. **위치**: `cs/{category}/images/` 폴더에 저장
3. **폰트**: `Arial, sans-serif` 사용 (크로스 플랫폼 호환)
4. **viewBox**: 컨텐츠 크기에 맞게 조정
5. **그라디언트**: 계층/중요도에 따라 색상 구분
6. **간선**: 노드보다 먼저 그리기 (노드 아래에 위치)
7. **텍스트 크기**: 타이틀 14px, 본문 11–12px, 레이블 9–10px
