# 에이전트 프롬프트 참조

test-coverage-check 스킬에서 사용하는 에이전트 디스패치 프롬프트 템플릿.

---

## review-test-coverage 에이전트 프롬프트

```
아래 변경된 Java 소스 파일들의 테스트 커버리지를 분석해주세요.

변경 파일 목록:
{파일 목록}

분석 항목:
1. 각 파일에 대응하는 테스트 파일 존재 여부
2. 테스트가 있는 경우: 주요 메서드별 테스트 존재 여부
3. 누락된 테스트 시나리오 목록 (경계값, 예외, 정상 케이스)
4. 우선순위별 정렬 (Critical > High > Medium > Low)

우선순위 기준:
- Critical: public API 메서드에 테스트 파일 자체가 없음
- High: 핵심 비즈니스 로직 메서드에 테스트 없음
- Medium: 예외/경계값 테스트 누락
- Low: 단순 getter/setter, 위임 메서드

결과를 아래 형식으로 정리:

| 소스 파일 | 테스트 파일 | 상태 | 누락 테스트 | 우선순위 |
|-----------|------------|------|------------|---------|
```

## test-generator 에이전트 프롬프트

```
아래 분석 결과를 기반으로 누락된 테스트를 생성해주세요.

커버리지 분석 결과:
{분석 결과 전문}

생성 규칙:
- Critical/High 우선순위 테스트만 생성
- 기존 테스트 파일이 있으면 해당 파일에 추가
- 기존 테스트 파일이 없으면 새로 생성
- 프로젝트의 테스트 컨벤션(JUnit5, AssertJ, Mockito 등) 준수
- @DisplayName으로 테스트 의도를 한글로 명시
- Given-When-Then 패턴 사용
- 생성 후 git add로 staging (commit은 하지 않음)
```

---

## 우선순위 판정 기준

| 우선순위 | 대상 | 예시 |
|---------|------|------|
| Critical | public API에 테스트 파일 자체 없음 | Controller 신규 생성, Service 핵심 메서드 |
| High | 핵심 비즈니스 로직 테스트 없음 | 금액 계산, 상태 전이, 권한 검증 |
| Medium | 예외/경계값 테스트 누락 | null 입력, 빈 리스트, 음수 값 |
| Low | 단순 위임/getter 테스트 없음 | DTO getter, 단순 조회 위임 |
